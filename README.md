TP 5 : Gestion de l'état et Structuration des Composants React
Ce projet porte sur la manipulation des données utilisateur via les formulaires et l'apprentissage de la communication entre composants, qu'elle soit directe (props) ou globale (Context API).
🎯 Objectifs d'apprentissage
Maîtrise des Formulaires : Apprendre la différence entre les approches "contrôlée" et "non-contrôlée".
Flux de données : Savoir comment faire circuler des informations entre un parent et ses enfants.
Gestion d'état global : Découvrir l'API Context de React pour éviter le "prop drilling" (faire passer des données à travers trop de niveaux de composants).
📋 Structure du TP (Étapes)
Initialisation : Création de l'environnement de travail React.
Formulaire Contrôlé : Liaison d'un input à un état React via l'attribut value et l'événement onChange.
Formulaire Non-Contrôlé : Utilisation du hook useRef pour récupérer les données au moment de la soumission (onSubmit).
Affichage et Structure : Organisation des composants pour afficher les données saisies de manière propre.
Context API : Mise en place d'un Provider pour centraliser certaines données de l'application.
